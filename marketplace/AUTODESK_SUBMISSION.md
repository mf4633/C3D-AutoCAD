# C3D Field Kit — Autodesk Marketplace submission blueprint

Turn the **C3D-AutoCAD** LISP library into a Civil 3D add-in for the
[Design and Make Marketplace](https://marketplace.autodesk.com/) (formerly App Store).

**Ground truth (repo):** 26 commands across 27 `.lsp` files (`_utils.lsp` has no command;
`acaddoc.lsp` / `lspload.lsp` are dev loaders — excluded from the bundle).

---

## Decisions locked 2026-07-23

- **List FREE on the Marketplace.** No PayPal publisher account, no entitlement
  API, lightest review. Autodesk supplies in-product discovery; the listing
  points at the GitHub repo, which carries the Gumroad buy link for the packaged
  kit. Rationale: distribution is the binding constraint, not price.

  `https://hydrocomplete.com` (switched off pe-calc.com, which has NO MX records

  domain differs from its company URL invites reviewer questions.
- **GUIDs are now real v4 GUIDs** (were hand-typed placeholders). `UpgradeCode`
  must never change again; `ProductCode` changes per released version.

## What Autodesk requires (Civil 3D path)

Civil 3D is an **AutoCAD vertical**. Follow the
[AutoCAD / vertical publisher guidelines](https://aps.autodesk.com/app-store/publisher-center/autocad-and-autocad-vertical):

| Requirement | How we satisfy it |
|-------------|-----------------|
| **Autoloader bundle** | `C3DFieldKit.bundle/` + `PackageContents.xml` |
| **Demand loading** | Each command is a `ComponentEntry` with `LoadOnCommandInvocation="True"` |
| **Ribbon UI** | Partial **CUIX** panel on the Plug-Ins tab (see `CUI_BUILD_NOTES.md`) |
| **Admin installer** | Inno Setup → `%ProgramData%\Autodesk\ApplicationPlugins\` |
| **Works after install** | No APPLOAD, no Support Path edits — autoloader handles everything |
| **Civil 3D compatibility** | Test on Civil 3D 2025–2027; declare versions in listing |
| **Help page** | `Help/quickstart.html` bundled + linked from `PackageContents.xml` |
| **Paid product** | PayPal publisher account ([setup guide](https://damassets.autodesk.net/content/dam/autodesk/www/adn/pdf/paypal-account-setup.pdf)) |

Publisher portal: [apps.autodesk.com → My Uploads](https://apps.autodesk.com/en/MyUploads)  
Questions: appsubmissions@autodesk.com

## Build steps (in order)

### 1. Generate the bundle

```powershell
powershell -ExecutionPolicy Bypass -File scripts/package-marketplace.ps1
```

This copies all command `.lsp` files into `Contents/`, keeps `bootstrap.lsp`, and
writes `PackageContents.xml` with all 26 `ComponentEntry` blocks.

### 2. Create ribbon CUIX

Follow `CUI_BUILD_NOTES.md`. Export **`C3DFieldKit.cuix`** into the bundle root
(next to `PackageContents.xml`). Autodesk requires at least one Plug-Ins tab panel.

### 3. Add icons

Download [icon publisher guidelines](https://www.autodesk.com/content/dam/autodesk/www/adn/pdf/icon-publishers.zip).
Place `Resources/icon.png` (256×256) and `Resources/icon_16.png` (16×16).

### 4. Test locally (no installer)

Copy the bundle to:

```
%ProgramData%\Autodesk\ApplicationPlugins\C3DFieldKit.bundle
```

Restart Civil 3D. Verify:

- Plug-Ins tab shows **C3D Field Kit** panel
- `LABELACRES`, `BD`, `BDTBL`, `PLT` run cold (first invoke loads the `.lsp`)
- No Support File Search Path or Trusted Location setup required

### 5. Build MSI/EXE installer

Open `marketplace/installer/C3DFieldKit.iss` in [Inno Setup](https://jrsoftware.org/isinfo.php),
point `SourceDir` at `dist/C3DFieldKit_v1`, compile. Test on a clean VM with Civil 3D.

### 6. Submit to Marketplace

Use the [app submission form](https://www.autodesk.com/content/dam/autodesk/www/adn/pdf/app-submission-marketing-technical-information.doc)
and [desktop submission overview PDF](https://damassets.autodesk.net/content/dam/autodesk/www/adn/pdf/desktop-app-submission-process-overview.pdf).

**Listing copy angles (mirror OpenRoads Field Kit):**

- **Name:** C3D Field Kit — 26 LISP Macros for Civil 3D
- **Subtitle:** Parcel labels, bearings tables, PNEZD-style coords, slope, layers, plot-all-PDF
- **Price:** Free (decided 2026-07-23) — the packaged kit with quick-start,
  cheat sheet and support is sold separately at
  https://fingazpdx.gumroad.com/l/bmdfn
- **Tags:** civil 3d, autocad, lisp, survey, parcel, cogo

## Civil 3D vs plain AutoCAD

These routines use **plain AutoLISP** (no Civil 3D API). They run in:

- Civil 3D (primary listing)
- AutoCAD / Map 3D (optional secondary compatibility — declare in listing)

Do **not** claim COGO alignment / parcel fabric integration you don't have.
`BD` / `BDTBL` operate on polylines, not Civil 3D parcels.
