# C3D Field Kit ‚Äî Autodesk Marketplace submission blueprint

Turn the **C3D-AutoCAD** LISP library into a Civil 3D add-in for the
[Design and Make Marketplace](https://marketplace.autodesk.com/) (formerly App Store).

**Ground truth (repo):** 26 commands across 27 `.lsp` files (`_utils.lsp` has no command;
`acaddoc.lsp` / `lspload.lsp` are dev loaders ‚Äî excluded from the bundle).

---

## Decisions locked 2026-07-23

- **List FREE on the Marketplace.** No PayPal publisher account, no entitlement
  API, lightest review. Autodesk supplies in-product discovery; the listing
  points at the GitHub repo, which carries the Gumroad buy link for the packaged
  kit. Rationale: distribution is the binding constraint, not price.
- **Support contact:** `support@hydrocomplete.com`, publisher URL
  `https://hydrocomplete.com` (switched off pe-calc.com, which has NO MX records
  at all). hydrocomplete.com already routes mail through ImprovMX with a valid
  SPF record, so no DNS work is needed ‚Äî **but confirm a `support@` alias or
  catch-all exists in the ImprovMX dashboard and send a live test**, because
  Autodesk emails this address during review.
- Publisher URL and support domain deliberately match; a listing whose contact
  domain differs from its company URL invites reviewer questions.
- **GUIDs are now real v4 GUIDs** (were hand-typed placeholders). `UpgradeCode`
  must never change again; `ProductCode` changes per released version.

## Build steps (in order)

### 1. Generate the ribbon CUIX

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-cuix.ps1
```

Drives `AcCui.dll` out-of-process ‚Äî **no CUI editor session required**. Writes
`marketplace/C3DFieldKit.bundle/C3DFieldKit.cuix`, then reopens it and asserts
macro / panel / row / button counts. See `CUI_BUILD_NOTES.md` for the layout
rules and the API traps.

### 2. Generate the bundle

```powershell
powershell -ExecutionPolicy Bypass -File scripts/package-marketplace.ps1
```

Copies all command `.lsp` files into `Contents/`, keeps `bootstrap.lsp`, picks up
the CUIX built in step 1, and writes `PackageContents.xml` with a
`ComponentEntry` per command (27 = 26 macros + the FIELDKIT about entry).

### 3. Add icons

Download [icon publisher guidelines](https://www.autodesk.com/content/dam/autodesk/www/adn/pdf/icon-publishers.zip).
Place `Resources/icon.png` (256√ó256) and `Resources/icon_16.png` (16√ó16).

### 4. Test locally (no installer)

Copy the bundle to:

```
%ProgramData%\Autodesk\ApplicationPlugins\C3DFieldKit.bundle
```

Restart Civil 3D. Verify:

- Plug-Ins tab shows **C3D Field Kit** panel
- `LABELACRES`, `BD`, `BDTBL`, `PLT` run cold (first invoke loads the `.lsp`)
- No Support File Search Path or Trusted Location setup required

### 5. Installer ‚Äî NOT required for submission

**Autodesk builds the MSI for you** from an autoloader `.bundle`. Submit the
zipped bundle (`dist/C3DFieldKit_v1_bundle.zip`) and skip this step.

`marketplace/installer/C3DFieldKit.iss` is kept only for local testing and the
Gumroad channel. Compiling it needs [Inno Setup](https://jrsoftware.org/isinfo.php),
which is **not installed on this machine** ‚Äî that is fine, nothing blocks on it.

### 6. Submit to Marketplace

Use the [app submission form](https://www.autodesk.com/content/dam/autodesk/www/adn/pdf/app-submission-marketing-technical-information.doc)
and [desktop submission overview PDF](https://damassets.autodesk.net/content/dam/autodesk/www/adn/pdf/desktop-app-submission-process-overview.pdf).

**Listing copy angles (mirror OpenRoads Field Kit):**

- **Name:** C3D Field Kit ‚Äî 26 LISP Macros for Civil 3D
- **Subtitle:** Parcel labels, bearings tables, PNEZD-style coords, slope, layers, plot-all-PDF
- **Price:** Free (decided 2026-07-23) ‚Äî the packaged kit with quick-start,
  cheat sheet and support is sold separately at
  https://fingazpdx.gumroad.com/l/bmdfn
- **Tags:** civil 3d, autocad, lisp, survey, parcel, cogo

## Civil 3D vs plain AutoCAD

These routines use **plain AutoLISP** (no Civil 3D API). They run in:

- Civil 3D (primary listing)
- AutoCAD / Map 3D (optional secondary compatibility ‚Äî declare in listing)

Do **not** claim COGO alignment / parcel fabric integration you don't have.
`BD` / `BDTBL` operate on polylines, not Civil 3D parcels.

## Pre-submission test matrix

Bundle is already staged at `%ProgramData%\Autodesk\ApplicationPlugins\C3DFieldKit.bundle`
(2026-07-23) ‚Äî the same path the installer targets. Restart Civil 3D to test.
Run the matrix against **2023 and 2026**, the two installed endpoints of the
declared range.

| Test | Pass criteria |
|------|---------------|
| Fresh Win 11 + C3D 2026 | Installer completes without errors |
| **CUIX loads** | "C3D Field Kit" tab appears; 26 buttons in 3 rows; no missing-CUIX error on startup |
| **Macro fires** | Clicking Label Acres runs `LABELACRES` (proves `MacroID` binding survived the save) |
| **Schema migration** | CUIX authored against the 2023 assembly loads clean in 2026 |
| First launch | Ribbon panel visible; no missing CUIX |
| `LABELACRES` on closed poly | Acres label at bbox center |
| `BDTBL` on lot polyline | Table inserts with bearings |
| `PLT` on saved DWG | PDF next to drawing |
| Uninstall | Bundle removed; Civil 3D starts clean |
| Upgrade | Install v1.1 over v1.0 without duplicate panels |
